# Route update after RH-115

The composite gate is monotone and contributes one new coarse `1e-8` support
record through the PSD packet block.  It also reveals a numerical-interface
requirement: exact directional actions cannot be mixed with a capacity upper
from another Gram assembly unless their operator mismatch has an outward
bound.

RH-116 should therefore optimize the memory depth using one internally
consistent assembly path.  It should measure the minimum depth needed by
each threshold, including the cost of a larger recent window and the decay
of the omitted positive tail.  This avoids transporting near-cancellation
data across independent binary64 constructions.
