# Exact centered local-kernel reference experiment

This directory contains a deliberately small, deterministic reference
implementation for the local kernel proposed in Paper 12. It is intended to
validate definitions and future optimized implementations. It is not designed
for production-scale boxes.

The only non-standard dependency is NumPy, used for full finite-matrix SVD.
All kernel construction and the exact double-centering certificate use the
Python standard library plus integer NumPy arrays.

## Mathematical convention

For

```text
I = [m_start, m_start + m_length)
J = [n_start, n_start + n_length)
P = {primes p: w < p <= y}
Q = product(P)
```

the surviving rows and columns satisfy

```text
gcd(m,Q) = gcd(n,Q) = 1.
```

For a nonzero even shift `h`, primes dividing `h` impose no shifted local
restriction on surviving rows and columns. Consequently the active prime set
and exact local mean are

```text
P_h     = {p in P: p does not divide h},
kappa_h = product over p in P_h of (p-2)/(p-1).
```

The binary and centered kernels on the survivor rectangle are

```text
K_h(m,n) = product over p in P_h of 1[(m*n+h) mod p != 0],
B_h(m,n) = K_h(m,n)/kappa_h - 1.
```

The code constructs `K_h` twice:

1. by checking each active prime separately;
2. by checking `gcd(m*n+h, product(P_h)) == 1`.

The run aborts unless these two complete finite matrices agree exactly.
Python integers make the direct-modulus construction safe from overflow.

## Raw and survivor normalization

The report deliberately keeps two normalizations. If `R` and `S` are the
numbers of surviving rows and columns, then

```text
raw_interval_normalized = sum(B_h)/(m_length*n_length)
survivor_normalized     = sum(B_h)/(R*S).
```

The first is normalized by the unsifted ambient box. It can look small merely
because the survivor support is sparse. The second measures the discrepancy
relative to the actual survivor rectangle. Neither quantity is an asymptotic
estimate.

The survivor matrix is also embedded into the ambient rectangle by putting
zero outside the coprime support. The JSON report gives complete singular-value
lists for:

- the raw survivor matrix;
- the ambient zero-extended matrix;
- the empirically double-centered survivor matrix.

These SVDs are ordinary floating-point NumPy/LAPACK computations. They are not
certified upper bounds for an infinite family of boxes.

## Exact double-centering

For an `R x S` binary matrix `K`, the integer matrix

```text
R*S*K - R*row_sum(K) - S*column_sum(K) + total_sum(K)
```

has row and column sums exactly zero. Multiplication by

```text
1/(kappa_h*R*S)
```

is exactly the double-centering of `K/kappa_h - 1`. The implementation stores
the integer numerator with Python integers and the scale as
`fractions.Fraction`; it verifies all zero margins before conversion to
floating point for SVD.

## Run

From this directory:

```bash
python reference_kernel.py \
  --m-start 100 \
  --m-length 24 \
  --n-start 140 \
  --n-length 20 \
  --w 3 \
  --y 19 \
  --h 2 \
  --output report.json
```

Negative nonzero even shifts are supported as well. The output is stable JSON
with exact rational values represented as strings and numerical SVD values as
JSON numbers. Runs use no stochastic sampling. The fixed seed
`20260713` is used only by the repeatable generated validation cases in the
test suite and is recorded in every report for future compatibility.

The program always prints an interpretation warning to standard error:

> Finite matrices and numerical singular values are diagnostics only; they do
> not prove a uniform subperiod-box estimate, an asymptotic theorem, or any
> statement about twin primes.

## Test

```bash
python -m unittest discover -s . -p 'test_*.py' -v
```

The tests cover:

- removal of every local factor with `p | h`;
- positive and negative general even shifts;
- exact agreement of the prime-loop and direct-modulus constructions;
- repeatability under a fixed validation seed;
- exact zero row and column sums after double-centering;
- both raw-interval and survivor normalizations;
- the known complete-period singular values for the one-prime case `p=5`;
- deterministic CLI output and the mandatory non-proof warning.

## Checked reference grid

The deterministic grid used for a compact manuscript table is generated by

```bash
python run_reference_grid.py
```

and checked without rewriting by

```bash
python run_reference_grid.py --check
```

The checked grid product is `data/reference-grid.csv`. Its first four rows use the
same two intervals and the same base modulus for `h=2,6,30,-6`. Therefore all
four rows have identical survivor counts. The active-prime and `kappa_h`
columns change with the shift: for example, `p=3` remains in the support
modulus for `h=6` but is absent from the shifted restriction and from
`kappa_6`. This is an explicit regression check of the inactive-coordinate
convention in the manuscript.

The CSV contains exact rational sums and both normalizations as well as
floating-point finite SVD diagnostics rounded to 12 significant digits to
avoid irrelevant last-bit differences between LAPACK backends. It has seven tiny cases and is a
definition/validation artifact, not a scaling experiment. Regenerate it only
after an intentional code or schema change, and commit the code and CSV
together.

The same command also writes `data/theorem-4.3-certificate.json`. This is an
exact rational brute-force regression check of the draft's subperiod
fully-active-shift completion theorem for

```text
Q = 3*5 = 15,  kappa = 3/8,  H = 23,
normalization = H/Q = 23/15.
```

The fixed rational coefficient sets deliberately include some nonunit points;
those points are removed from both the bilinear form and the `L_alpha,L_beta`
norms exactly as in the theorem. The certificate enumerates the short shift
interval and every fully active residue in a complete period, then checks

```text
abs(short_energy - (H/Q)*complete_energy)
    <= E(Q)*(L_alpha*L_beta)^2
```

with `fractions.Fraction` arithmetic. In particular it regression-tests
`H/Q`, not `H/phi(Q)`. The JSON contains the exact discrepancy, bound, and
nonnegative slack. This finite verification catches normalization or
implementation regressions; it is not a proof of Theorem 4.3.

## Scope boundary

This reference program can find a finite counterexample to a proposed explicit
finite inequality, expose a residual-class mode, and validate optimized code.
It cannot test all bounded coefficient sequences, prove uniformity over box
locations or aspect ratios, transfer an averaged-shift observation to `h=2`,
or turn a polylogarithmic-cutoff experiment into a polynomial-cutoff theorem.
